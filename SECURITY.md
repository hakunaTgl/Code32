# Security Policy

## Supported Versions

Currently supported versions of Code32:

| Version | Supported          |
| ------- | ------------------ |
| main    | :white_check_mark: |

## Reporting a Vulnerability

If you discover a security vulnerability in Code32, please report it by:

1. **DO NOT** open a public issue
2. Email the repository owner directly through GitHub
3. Include detailed information about the vulnerability:
   - Description of the issue
   - Steps to reproduce
   - Potential impact
   - Suggested fixes (if any)

## Security Best Practices

When using Code32:

- Keep dependencies up to date
- Review code changes carefully
- Follow Python security best practices
- Use virtual environments for isolation
- Never commit sensitive data (API keys, passwords, etc.)

## Response Timeline

- Initial response: Within 48 hours
- Status updates: Every 5-7 days
- Resolution: Depends on severity and complexity

## Disclosure Policy

We follow responsible disclosure practices:

- Security issues will be patched before public disclosure
- Credit will be given to reporters (unless anonymity is requested)
- Public disclosure will occur after fixes are released
